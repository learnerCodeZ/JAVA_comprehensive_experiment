# 智能家居控制系统 - 功能手册

## 快速开始

### 编译

```bash
javac -d out -sourcepath src -cp "lib/*" src\com\smarthome\Main.java
```

### 运行

```bash
java -cp "out;lib/*" com.smarthome.Main
```

### 登录账号

| 用户名 | 密码 | 角色 | 可用功能 |
|--------|------|------|---------|
| admin | admin123 | 管理员 | 全部功能：设备增删改查、场景模式、用户管理、日志导出、网络控制、系统配置 |
| user | user123 | 普通用户 | 查看设备、控制开关、查看日志 |

## 功能一览

### 1. 用户登录

程序启动后弹出登录对话框，输入用户名和密码登录。登录失败会提示"用户名或密码错误"。

### 2. 主界面布局

```
┌──────────────────────────────────────────────────────┐
│  文件(F)  设备(D)  场景(S)  帮助(H)  网络(N)          │
├─────────┬────────────────────────┬──────────┤
│ 房间列表 │       设备卡片网格       │ 设备控制  │
│         │  [L]    [A]    [C]     │          │
│  全部    │ 客厅灯  客厅空调 客厅窗帘 │ 全部开启  │
│  客厅    │  [L]    [A]    [S]     │ 全部关闭  │
│  卧室    │ 卧室灯  卧室空调 智能音箱 │          │
│  厨房    │                        │          │
│  书房    │                        │          │
├─────────┴────────────────────────┴──────────┤
│  操作日志                            [导出日志] │
│  21:30:05 - 开启 客厅灯                       │
│  21:30:12 - 关闭 卧室空调                      │
└──────────────────────────────────────────────┘
```

- **左侧** — 点击房间名筛选设备，点"全部"显示所有
- **中央** — 设备卡片，开启时金色边框+深色背景，关闭时灰色边框+白色背景
- **右侧** — 全部开启/全部关闭按钮
- **底部** — 操作日志 + 导出日志按钮

### 3. 设备控制

点击任意设备卡片，弹出控制对话框：

| 操作 | 说明 |
|------|------|
| 开启 | 开启该设备 |
| 关闭 | 关闭该设备 |
| 关闭窗口 | 不操作，关闭对话框 |

**4种设备类型及参数**：

| 设备 | 图标 | 特有参数 | 参数范围 |
|------|------|---------|---------|
| 智能灯 | [L] | 亮度、颜色 | 亮度 0-100% |
| 空调 | [A] | 温度、模式 | 温度 16-30°C，模式：制冷/制热/除湿等 |
| 智能窗帘 | [C] | 开合度 | 0-100%（0=全关，100=全开） |
| 智能音箱 | [S] | 音量、播放状态 | 音量 0-100 |

### 4. 场景模式（仅管理员）

通过菜单 **场景(S)** 触发，一键控制多个设备：

| 场景 | 效果 |
|------|------|
| 回家模式 | 灯全开(亮度100%) + 空调开(26°C制冷) + 窗帘全开 |
| 离家模式 | 所有设备关闭 |
| 睡眠模式 | 灯全关 + 空调开(25°C制冷) + 窗帘全关 + 音箱开(音量10) |

### 5. 设备管理（仅管理员）

通过菜单 **设备(D)** 操作：

| 功能 | 操作方式 |
|------|---------|
| 添加设备 | 选择类型 → 输入名称 → 选择房间 → 确定 |
| 删除设备 | 从下拉列表选择设备 → 确认删除 |

### 6. 网络远程控制

通过菜单 **网络(N) → 网络控制** 打开网络面板：

**服务端控制**：
| 按钮 | 说明 |
|------|------|
| 启动服务 | 在 8888 端口启动 Socket 服务端 |
| 停止服务 | 停止服务端 |

**客户端测试**：在命令输入框输入命令，点击"发送"测试

| 命令 | 说明 | 示例 |
|------|------|------|
| `TURN_ON:设备ID` | 远程开启设备 | `TURN_ON:d1` |
| `TURN_OFF:设备ID` | 远程关闭设备 | `TURN_OFF:d3` |
| `GET_ALL` | 获取所有设备状态 | `GET_ALL` |

响应格式：`OK:操作结果` 或 `ERROR:错误信息`

### 7. 操作日志

- 底部日志区自动显示最近 20 条操作记录
- 格式：`HH:mm:ss - 操作 设备名`
- **导出日志**（仅管理员）：点击"导出日志"按钮，选择保存路径，导出为 txt 文件

### 8. 数据持久化

- 设备和用户数据自动保存到 JSON 文件（`data/devices.json`、`data/users.json`）
- 关闭程序后再次启动，设备状态和用户数据保留
- 首次运行自动创建示例数据：4个房间 + 6个设备

### 9. 权限控制

| 功能 | 管理员 | 普通用户 |
|------|--------|---------|
| 查看设备 | ✅ | ✅ |
| 控制设备开关 | ✅ | ✅ |
| 全部开启/关闭 | ✅ | ✅ |
| 添加设备 | ✅ | ❌ |
| 删除设备 | ✅ | ❌ |
| 场景模式 | ✅ | ❌ |
| 导出日志 | ✅ | ❌ |
| 网络控制 | ✅ | ✅ |

普通用户登录时，无权限的菜单项和按钮自动置灰不可点击。

## 配置文件

`config/config.properties` 可修改系统参数：

```properties
monitor.interval=5000    # 监控轮询间隔（毫秒）
data.directory=data      # 数据存储目录
config.directory=config  # 配置文件目录
max.log.count=100        # 最大日志条数
server.port=8888         # 网络服务端口
```

## 技术知识点

| 知识点 | 对应实现 |
|--------|----------|
| 类与对象 | Device、Room、User 等实体类 |
| 继承 | Light、AirConditioner、Curtain、Speaker 继承 Device |
| 多态 | 统一接口控制不同设备，场景模式中 instanceof 判断 |
| 抽象类 | Device 抽象基类（含抽象方法 control()、getInfo()） |
| 接口 | Controllable（控制行为）、Monitorable（监控行为） |
| 异常处理 | DeviceException 自定义异常（含 errorCode） |
| 包结构 | entity/interfaces/service/dao/ui/network/factory/config/util 分层 |
| 多线程 | MonitorService 守护线程定时监控 |
| GUI编程 | Swing 组件：JFrame、JDialog、JList、JPanel、JTextArea 等 |
| 网络通信 | Socket 服务端/客户端（TCP 文本协议） |
| 文件存储 | JSON 持久化（Gson 库序列化/反序列化） |
| 配置文件 | Properties 文件 + 单例模式 AppConfig |
| 设计模式 | 工厂模式（DeviceFactory）、单例模式（AppConfig） |
